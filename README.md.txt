Version Control: This is a type of software that enables developers to be able to safekeep the changes that have been made to a source code in the process of software development. This makes it possible to recall the source code and all the changes that might have been made as softwares and applications undergoe continuous development. They are also called Source Code Management tools,

Git: This is a version control system that helps a developer collate and organize the source code history, from the inception, through the the different processes of development.

Process of commiting a project to github repository: A repository that includes a named project on GitHub is created, followed by the creation of a folder on the desktop, with the same name of that of the repository on GitHub. The terminal on the code text editor is then opened. Firstly, the git is initialized by inputting ''git init'' to make sure that git is recognised by the code text editor. The index.html file that contains what is to be pushed is includded by ''git add .''. Then ''git commit-m''first commit'', followed by git branch-M main. The git remote repository is linked to the local repository by inputting ''git remote add origin ''particular address''. To push, ''git push -u origin main'' is inputted.
